# 阿里云Alibaba Cloud Linux 3.2104 LTS部署考试系统安装手册

## 一、准备工作

### 1. 本地准备
- 确保本地Windows 11开发环境已完成项目开发
- 项目代码已提交到版本控制系统（如GitHub、GitLab等）
- 准备服务器登录凭证（用户名和密码/密钥对）

### 2. 阿里云服务器配置
- 确保阿里云服务器已购买并启动
- 操作系统：Alibaba Cloud Linux 3.2104 LTS 64位
- 安全组已开放80、443端口（HTTP/HTTPS）
- 建议配置：2核4G以上内存，50G以上磁盘空间

## 二、服务器环境配置

### 1. 登录服务器
```bash
# 使用SSH密钥登录（推荐）
ssh -i your-key.pem root@服务器公网IP

# 或使用密码登录
ssh root@服务器公网IP
```

### 2. 更新系统
```bash
sudo dnf update -y
sudo dnf upgrade -y
```

### 3. 安装必要依赖
```bash
sudo dnf install -y git curl wget unzip gcc gcc-c++ make
```

### 4. 安装Docker
```bash
# 安装Docker
sudo dnf install -y docker-ce docker-ce-cli containerd.io

# 启动Docker服务
sudo systemctl start docker
sudo systemctl enable docker

# 验证Docker安装
docker --version
```

### 5. 安装Docker Compose
```bash
# 下载Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 赋予执行权限
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

### 6. 配置Docker用户组（可选）
```bash
# 创建docker用户组
sudo groupadd docker

# 将当前用户加入docker组
sudo usermod -aG docker $USER

# 重新登录生效
exit
ssh root@服务器公网IP
```

## 三、项目部署

### 1. 克隆项目代码
```bash
# 克隆项目
git clone https://github.com/your-repo/exam-system.git

# 进入项目目录
cd exam-system
```

### 2. 配置环境变量
编辑 `docker-compose.yml` 文件：
```bash
# 编辑配置文件
vim docker-compose.yml
```

修改以下配置：
- `SECRET_KEY`：设置为随机字符串
- `SQLALCHEMY_DATABASE_URI`：可保持默认，或修改为其他数据库
- 其他配置根据需要调整

### 3. 创建必要目录
```bash
# 创建nginx配置目录和SSL目录
mkdir -p nginx/ssl

# 创建上传目录
mkdir -p uploads

# 赋予权限
sudo chmod -R 755 uploads
```

### 4. 配置Nginx
编辑 `nginx/nginx.conf` 文件：
```bash
# 编辑Nginx配置
vim nginx/nginx.conf
```

修改以下内容：
- `server_name`：设置为您的域名或服务器IP
- SSL证书配置（如果使用HTTPS）

### 5. 配置SSL证书（可选）
如果需要HTTPS访问，在 `nginx/ssl` 目录放置证书文件：
- `cert.pem`：SSL证书
- `key.pem`：私钥

### 6. 构建并启动服务
```bash
# 构建并启动服务
sudo docker-compose up -d --build
# 查看服务状态
sudo docker-compose ps
```

### 7. 检查服务日志
```bash
# 查看后端日志
sudo docker-compose logs -f backend

# 查看Nginx日志
sudo docker-compose logs -f nginx
```

## 四、服务启动与验证

### 1. 开放端口
```bash
# 查看防火墙状态
sudo systemctl status firewalld

# 开放80和443端口
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --reload

# 查看开放的端口
sudo firewall-cmd --list-all
```

### 2. 验证服务访问
在浏览器中访问：
- 前端：`http://服务器公网IP` 或 `https://服务器公网IP`
- 后端API：`http://服务器公网IP/api` 或 `https://服务器公网IP/api`

### 3. 测试系统功能
- 注册新用户
- 登录系统
- 导入题目（PDF/Word/Excel）
- 创建试卷
- 开始考试
- 查看成绩

## 五、常见问题排查

### 1. 服务启动失败
```bash
# 查看详细日志
sudo docker-compose logs backend

# 检查端口是否被占用
sudo netstat -tuln | grep 5000
sudo netstat -tuln | grep 80
sudo netstat -tuln | grep 443
```

### 2. 数据库连接问题
```bash
# 检查Redis服务状态
sudo docker-compose logs redis

# 检查数据库文件权限
sudo ls -la uploads/
```

### 3. 文件上传问题
```bash
# 检查上传目录权限
sudo chmod -R 755 uploads
```

### 4. Nginx配置问题
```bash
# 检查Nginx配置
sudo docker-compose exec nginx nginx -t

# 重启Nginx
sudo docker-compose restart nginx
```

### 5. SSL证书问题
如果使用自签名证书，浏览器会显示安全警告，建议在生产环境使用正规SSL证书。

```bash
sudo docker-compose down
```

### 2. 重启服务
```bash
sudo docker-compose restart
```

### 3. 更新代码
```bash
# 拉取最新代码
git pull

# 重新构建并启动
sudo docker-compose up -d --build
```

### 4. 备份数据
```bash
# 备份数据库
sudo cp uploads/exam_system.db /path/to/backup/

# 备份上传文件
sudo cp -r uploads/* /path/to/backup/uploads/
```

### 5. 查看容器状态
```bash
sudo docker ps -a
```

### 6. 查看系统资源使用情况
```bash
# 查看CPU和内存使用
top

# 查看磁盘使用
df -h
```

## 七、生产环境优化建议

1. **使用域名**：将域名解析到服务器IP，并在Nginx配置中设置
2. **使用正规SSL证书**：推荐使用Let's Encrypt等免费证书
3. **配置定期备份**：设置crontab定时任务备份数据库和上传文件
4. **启用监控**：安装监控工具如Prometheus和Grafana监控服务状态
5. **安全加固**：
   - 关闭不必要的端口
   - 配置防火墙规则
   - 定期更新系统和依赖
   - 使用密钥登录，禁用密码登录
6. **性能优化**：
   - 根据服务器配置调整Docker Compose中的资源限制
   - 配置Redis持久化
   - 优化Nginx配置，启用gzip压缩

## 八、故障恢复

### 1. 服务无法启动
```bash
# 检查Docker服务状态
sudo systemctl status docker

# 重启Docker服务
sudo systemctl restart docker

# 重新启动服务
sudo docker-compose up -d
```

### 2. 数据丢失
```bash
# 从备份恢复
sudo cp /path/to/backup/exam_system.db uploads/
sudo cp -r /path/to/backup/uploads/* uploads/

# 重启服务
sudo docker-compose restart
```

### 3. 服务器重启后服务未自动启动
```bash
# 确保Docker服务自启动
sudo systemctl enable docker

# 配置服务自启动
# 在/etc/rc.d/rc.local中添加启动命令
echo "cd /path/to/exam-system && sudo docker-compose up -d" >> /etc/rc.d/rc.local
sudo chmod +x /etc/rc.d/rc.local
```
