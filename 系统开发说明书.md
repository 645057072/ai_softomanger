## 技术架构

### 部署架构
- **容器化**: Docker
- **服务编排**: Docker Compose
- **反向代理**: Nginx
- **应用服务器**: Gunicorn

## 常见问题

### Nginx 403 错误
**原因**: 静态文件目录为空或权限问题  
**解决**: 创建默认 index.html 并确保正确挂载

### 容器间网络通信
**配置**: 使用专用 Docker 网络 exam-network  
**通信**: 使用服务名进行容器间通信

### 路由守卫
**实现**: 在 router/index.js 中配置 beforeEach 守卫  
**逻辑**: 检查 token，决定路由跳转

#### 6. 自动化部署脚本开发

**功能需求**:
- 自动提交代码到 Git 仓库
- 自动连接到阿里云服务器
- 自动拉取最新代码
- 自动重启 Docker 服务
- 自动验证部署状态

**技术实现**:
- 创建 PowerShell 脚本（Windows 环境）
- 创建 Bash 脚本（Linux 环境）
- 使用 SSH 连接服务器
- 使用 Git 进行代码同步
- 使用 Docker Compose 管理服务

**修改文件**:
- `auto-deploy.ps1` - Windows 自动化部署脚本
- `auto-deploy.sh` - Linux 自动化部署脚本

**技术要点**:
- SSH 自动连接（使用 sshpass 或手动输入密码）
- Git 自动提交和推送
- Docker 服务自动重启
- 部署状态自动验证

**使用方法**:
```powershell
# Windows 环境
.\auto-deploy.ps1

# Linux 环境
bash auto-deploy.sh
```

**部署流程**:
1. 检查代码变更并提交到 Git
2. 推送到远程 Git 仓库
3. SSH 连接到阿里云服务器
4. 拉取最新代码
5. 停止现有 Docker 服务
6. 重新构建并启动服务
7. 验证容器状态和日志

如果自动化脚本无法使用，可以按照以下步骤手动部署：

#### 1. 提交代码到 Git
```bash

git push origin main
```

#### 2. 连接到服务器
```bash
ssh root@47.93.44.247
```

#### 3. 拉取代码
```bash
cd /opt/ai_softomanger
git pull origin main
```

#### 4. 重启服务
```bash
docker-compose down
docker-compose up -d --build
```

#### 5. 查看日志
```bash

```
