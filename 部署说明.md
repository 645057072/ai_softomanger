## 阿里云服务器部署步骤

### 1. 配置 Docker 镜像加速器（必须）

```bash
# 创建 Docker 配置目录
sudo mkdir -p /etc/docker

# 创建 Docker 配置文件（使用多个镜像加速器）
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://docker.1panel.live",
    "https://hub.rat.dev"
  ],
  "max-concurrent-downloads": 10,
  "log-driver": "json-file",
  "log-level": "warn",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
EOF

# 重启 Docker 服务
sudo systemctl daemon-reload
sudo systemctl restart docker

# 验证配置是否生效
docker info | grep -A 10 "Registry Mirrors"
```

### 2. 上传代码到服务器

```bash
# 方法 1: 使用 git
cd /opt
git clone https://github.com/你的仓库/ai_softomanger.git
cd ai_softomanger

# 方法 2: 使用 scp 上传（在本地执行）
# scp -r ./* root@47.93.44.247:/opt/ai_softomanger

# 方法 3: 使用 rsync（在本地执行）
# rsync -avz --progress ./* root@47.93.44.247:/opt/ai_softomanger/
```

### 3. 构建和启动服务

```bash
cd /opt/ai_softomanger

# 停止旧服务（如果有）

# 重新构建和启动
docker-compose up -d --build
```

## 常见问题排查

### 1. 后端服务无法启动

```bash
# 查看后端详细日志
docker logs exam-system-backend --tail 100

# 进入容器检查
docker exec -it exam-system-backend bash

# 检查数据库文件
ls -lh /app/exam_system/exam_system.db
```

### 2. 前端无法访问

```bash
# 检查前端构建是否成功
docker logs exam-system-frontend

# 检查 Nginx 配置
docker exec exam-system-nginx cat /etc/nginx/nginx.conf

# 检查前端文件是否存在
docker exec exam-system-nginx ls -lh /usr/share/nginx/html
```

### 3. 网络连接问题

```bash
# 检查容器网络
docker network ls

# 检查容器间通信
docker exec exam-system-nginx ping backend

# 检查端口监听
docker exec exam-system-backend netstat -tlnp
```

### 4. 数据库问题

