## 部署脚本说明

本项目提供了自动化部署脚本，可以一键完成代码提交、推送到 GitHub、连接服务器、部署等操作。

## 前置要求

### 本地环境
- 已安装 Git
- 已安装 OpenSSH 客户端（Windows 10 自带）
- 已配置 Git 远程仓库

### 服务器环境
- 已安装 Git
- 已安装 Docker 和 Docker Compose
- 已配置 SSH 访问

## 部署脚本

### Windows 环境（推荐）

使用 `deploy.ps1` 脚本：

```powershell
# 在项目根目录运行
.\deploy.ps1
```

**功能特点**：
- 自动检测代码变更
- 自动提交到 Git
- 自动推送到 GitHub
- 自动上传 `frontend/dist` 到服务器
- SSH 登录服务器后自动 `git pull`，并执行仓库内脚本 **`scripts/docker-compose-redeploy.sh`**（内部为 `docker compose` 或 **`sudo docker compose up -d --build`**，无需手工登录服务器执行 compose）

**GitHub Actions**：向 `master`/`main` 推送后，工作流会 rsync 代码到服务器并同样调用上述脚本完成重建。
- 自动验证部署状态

**参数选项**：
```powershell
# 跳过代码提交步骤（仅部署）
.\deploy.ps1 -SkipCommit
```

### Linux 环境

使用 `auto-deploy.sh` 脚本：

```bash
# 在项目根目录运行
bash auto-deploy.sh
```

## 配置文件

### 配置模板

项目提供了配置文件模板 `deploy.config.example`：

```bash
# Git 仓库配置
GIT_REPO_URL="github.com:645057072/ai_softomanger.git"
GIT_BRANCH="master"

# 服务器配置
SERVER_IP="47.93.44.247"
SERVER_USER="root"
SERVER_PORT="22"
# 说明：请使用 SSH 私钥免密登录（在服务器上配置 authorized_keys）

# 部署路径
REMOTE_PATH="/opt/ai_softomanger"

# 服务配置
SERVICE_PORT="80"
```

### 使用配置文件

1. 复制模板文件：
```bash
cp deploy.config.example deploy.config
```

2. 编辑配置文件，填写实际配置：
```bash
nano deploy.config
```

3. 修改脚本以使用配置文件（可选）

**注意**：`deploy.config` 已添加到 `.gitignore`，不会提交到 Git。

## 部署流程

### 自动化部署流程

1. **代码提交**
   - 检测未提交的更改
   - 自动添加并提交
   - 推送到 GitHub

2. **服务器连接**
   - SSH 连接到阿里云服务器
   - 验证连接状态

3. **代码拉取**
   - 切换到部署目录
   - 拉取最新代码

4. **服务重启**
   - 停止现有服务
   - 重新构建镜像
   - 启动所有服务

5. **状态验证**
   - 检查容器状态
   - 查看服务日志

### 手动部署流程

如果自动化脚本无法使用，可以手动部署：

#### 1. 提交代码
```bash
git add .
git commit -m "提交说明"
git push origin master
```

#### 2. 连接到服务器
```bash
ssh root@47.93.44.247
```

#### 3. 拉取代码
```bash
cd /opt/ai_softomanger
git pull origin master
```

#### 4. 重启服务
```bash
docker-compose down
docker-compose up -d --build
```

#### 5. 查看日志
```bash
docker-compose logs -f
```

## 常见问题

### SSH 连接失败

**问题**：无法连接到服务器

**解决方案**：
1. 检查网络连接
2. 检查服务器 IP 地址
3. 检查 SSH 服务是否运行
4. 检查防火墙设置

### Git 推送失败

**问题**：无法推送到 GitHub

**解决方案**：
1. 检查 Git 远程仓库配置
2. 检查网络连接
3. 检查 GitHub 访问权限
4. 检查分支名称

### Docker 服务启动失败

**问题**：Docker 服务无法启动

**解决方案**：
1. 查看容器日志：`docker-compose logs`
2. 检查端口占用
3. 检查配置文件
4. 重新构建镜像：`docker-compose build`

### 代码拉取失败

**问题**：服务器上无法拉取代码

**解决方案**：
1. 检查 Git 配置
2. 检查网络连接
3. 检查仓库访问权限
4. 手动执行 `git pull` 查看错误信息
